¿Qué es Angular?

Angular es software de desarrollo web de código abierto creado en 2010 y mantenido por Google. Angular combina HTML, CSS y JavaScript (o TypeScript) para realizar interfaces de usuario modernas y fáciles de escalar.

Una de las principales características de Angular es su enfoque basado en componentes. Los componentes son estructuras reutilizables que encapsulan el comportamiento o el diseño de una parte de la interfaz de usuario. Esto permite a los programadores dividir aplicaciones complejas partes más manejables y más cómodadas de modificar.

En resumen, Angular es una herramienta versátil y poderosa para desarrolladores que buscan construir aplicaciones web robustas y modernas, con una gran escalabilidad.

¿Por qué elegir Angular?

Angular es una elección habitual entre desarrolladores y empresas de software debido a su potencia para el desarrollo de aplicaciones web. Una de las razones principales para elegir Angular es que incluye herramientas de enrutamiento, gestión de formularios, servicios HTTP y más.

Otra ventaja de Angular es que al estar supervisado por Google, se garantizan las actualizaciones periódicas, nuevas funcionalidades y compatibilidad con otras aplicaciones de desarrollo web. Grandes empresas como Google, Microsoft y Amazon utilizan Angular para el desarrollo de sus webs públicas o para los frot-end de sus aplicativos (Google Analytics, Google Adwrods, etc). Esto demuestras que Angular es muy válido para desarrollar proyectos complejos y de gran envergadura.

Para las empresas de software es una herramienta de desarrollo muy interesante porque facilita la reutilización de componentes en distintos proyectos, ahorrando tiempos y esfuerzos.

Ventajas y desventajas

Las principales ventajas de Angular son:

  • Desarrollo de código limpio
  • Fácil de construir, mantener, probar y actualizar
  • Compatibilidad con HTTP, AJAX, etc
  • Representación rápida del lado del servidor
  • Compatibilidad con vistas sin renderizado del lado del navegador
  • Los componentes que se ejecutan en paralelo reducen la cantidad de líneas necesarias para el código
  • Productividad mejorada para los desarrolladores
  • Se mejora el rendimiento de la aplicación.
  • Angular cuenta con una robusta herramienta de CLI (Command Line Interface) que simplifica tareas comunes como la generación de componentes, volcado a repositorios, etc.

    • Algunas desventajas de Angular son:

  • Los principiantes se enfrentan a una curva de aprendizaje muy pronunciada.
  • Los paquetes de gran tamaño pueden afectar el tiempo de carga de la aplicación
  • Estructura sintáctica compleja
  • Flexibilidad limitada en comparación con React
  • Se utiliza mucho TypeScript
  • Actualizar a versiones más nuevas puede ser difícil
  • Tiempo de renderizado más lento en comparación con React

    • Angular vs React

    Angular y React son dos de las tecnologías más populares para el desarrollo web, pero tienen diferencias significativas. Angular ofrece todas las herramientas necesarias para desarrollar una aplicación desde cero. React, por el contrario es una biblioteca enfocada en la interfaz de usuario, lo que significa que requiere integración con otras herramientas para funcionalidades como el enrutamiento o la gestión de estados.

    En cuanto al lenguaje, Angular utiliza TypeScript de forma predeterminada, lo que mejora la seguridad y la mantenibilidad del código. React permite usar JavaScript puro, pero también es compatible con TypeScript para los desarrolladores que deseen un tipado estático. La elección entre ambos depende en gran medida de las preferencias y necesidades del equipo.

    En cuanto al rendimiento, React suele ser más ligero y rápido debido a su enfoque en la biblioteca central y su uso del DOM virtual. Angular, aunque más pesado, ofrece herramientas para optimizar el rendimiento, como la renderización del lado del servidor y la división de código.

    En conclusión, Angular es ideal para proyectos que requieren una solución completa y estructurada, mientras que React es más adecuado para desarrolladores que buscan flexibilidad y simplicidad en el desarrollo de interfaces.

    Si partimos de cero en ambos casos hay que tener en cuenta la facilidad de aprendizaje de React frente a Angular.

    TypeScript

    TypeScript es un superconjunto de JavaScript desarrollado por Microsoft que añade tipado estático y otras funcionalidades avanzadas al lenguaje. Es el lenguaje principal de Angular y proporciona una base sólida para escribir código más seguro. TypeScript se convierte en JavaScript compatible con navegadores y entornos de ejecución.

    El uso de TypeScript en Angular permite detectar errores en tiempo de compilación en lugar de tiempo de ejecución, lo que reduce el riesgo de fallos en producción.

    En resumen, TypeScript es una elección ideal para el desarrollo con Angular, ya que proporciona una base sólida, reduce errores y mejora la experiencia general de desarrollo.

    ¿Qué aplicaciones tiene?

    Angular es ampliamente utilizado para desarrollar aplicaciones web modernas y robustas como por ejemplo:

  • Aplicaciones web empresariales con requisitos complejos: herramientas internas, paneles de administración y sistemas de gestión empresarial (ERP), etc.
  • Proyectos a gran escala con múltiples equipos
  • Aplicaciones de datos en tiempo real, como chats y sistemas de colaboración.
  • Aplicaciones web progresivas (PWA)
  • Solicitudes de una sola página (SPA)
  • Aplicaciones con requisitos de alto rendimiento
  • Aplicaciones móviles multiplataforma e híbridas
  • Aplicaciones con un fuerte enfoque en pruebas y mantenibilidad.
  • Aplicaciones de comercio electrónico, donde se requiere una interfaz rápida y atractiva para los usuarios.

    • ¿Qué prerrequisitos necesita?

    Para comenzar a trabajar con Angular, es importante tener conocimientos básicos de desarrollo web. Esto incluye familiaridad con HTML, CSS,JavaScript y TypeScript, ya que son los pilares fundamentales para construir aplicaciones con Angular. Entender conceptos como el DOM (Document Object Model) y la programación orientada a objetos también es beneficioso.

    Recomendable el conocimiento en herramientas de control de versiones como Git, ya que la mayoría de los proyectos Angular se gestionan en entornos colaborativos. Familiarizarse con npm (Node Package Manager) también es importante ya que se utiliza para instalar Angular CLI y otras funcionalidades.

    ¿Sobre qué lenguajes se apoya?

    Como hemos comentado en los otros apartados se basa principalmente en:

  • TypeScript, que permite a los desarrolladores escribir código compatible con las versiones más recientes de JavaScript.
  • HTML también es un lenguaje fundamental en Angular, ya que se utiliza para definir la estructura y el diseño de las vistas.
  • CSS es otro pilar en Angular, ya que se utiliza para el diseño de la web final.

    • Instalación de Angular

    Angular se puede instalar tanto en Windows como en Linux y macOS. En todos los casos se requieren una serie de pasos previos de configuración e instalación para su correcto funcionamiento.

    Instalación en Windows

    Paso 1: Instalación de Node.js y npm

    Lo primero que tenemos que instalar es el Node.js y npm (Node Package Manager) en nuestra máquina. Podemos descargarlo desde la web oficial:

    El instalador de Node.js también instala npm con lo que no será neceario realizar otra instalación.

    Podemos seguir el proceso de instalación que viene por defecto. Al finalizar podemos abrir una consola para comprobar si se ha instalado correctamente utilizando los comandos:

    					Node –version → comprobará Node.js 
					npm –version → comprobará npm

    Paso 2: Instalar Angular CLI

    Una vez realizado el Paso 1 (con la comprobación de las versiones instaladas) estamos en disposición de instalar el paquete de Angular. Para ello ejecutamos también desde la consola el siguiente comando:

    					npm install -g @angular/cli

    Este comando instala la última versión estable de la herramienta CLI de Angular y hace que esté disponible para su uso en nuestro ordenador. Al igual que hicimos en el paso 1 verificamos la instalación con el comando “ng –version”.

    Instalación en Linux

    En Linux, se necesita instalar Node.js y npm. Esto puede hacerse mediante el gestor de paquetes de la distribución que usted tenga en su equipo, "sudo apt install nodejs npm". Es posible que se necesite agregar repositorios específicos para obtener versiones más recientes.

    Es importante verificar que Node.js y npm estén instalados correctamente utilizando "node -v" y "npm -v" en la consola. Para instalar Angular CLI utilizaremos el comando "npm install -g @angular/cli".

    Tenga especial cuidado con los permisos al instalar los paquetes necesarios ya que es un error común.

    Instalación en macOS

    En macOS, también es necesario instalar Node.js y npm. La instalación puede hacerse mediante el instalador oficial desde el sitio web de Node.js

    Después de instalar Node.js, se recomienda verificar las versiones instaladas mediante los comandos "node -v" y "npm -v" en el terminal de su mac.

    Instalación de Angular CLI: al igual que en Windows, Angular CLI se instala ejecutando "npm install -g @angular/cli" en la consola de comandos.

    Configuración adicional: En algunos casos, es útil configurar el shell de macOS para reconocer el comando ng correctamente, especialmente si se utilizan herramientas como Zsh o Bash.

    Estructuras de carpetas

    Cuando se crea un proyecto de Angular se crean una serie de carpetas que a continación vamos a analizar:

  • .angular: Se generada automáticamente por Angular CLI en el momento de ejecutar la sentencia de creación de proyecto “ng serve”. Se utiliza para almacenar archivos temporales y de configuración interna relacionados con la compilación y el desarrollo. Por ejemplo, aquí se guardan los ficheros generados por Angular durante la ejecución de comandos como ng build o ng serve.
  • .vscode: Contiene configuraciones específicas del editor Visual Studio Code para el proyecto Angular. Se va a crear estemos utilizando o no el editor Visual Code, Estas configuraciones pueden incluir extensiones utilizadas, atajos de teclado personalizados, ajustes del depurador o configuraciones de formateo del código. Es útil para compartir el entorno de desarrollo entre los miembros del equipo de desarrollo.
  • node_modules:: Es donde se guardan las dependencias de Node.js necesarias para el proyecto Angular. Cuando se ejecuta el comando npm install, este directorio se llena con los paquetes descargados de npm, incluyendo Angular. Esta carpeta no debe ser modificada manualmente y tampoco se sube al repositorio, ya que su contenido se puede regenerar usando npm install.
  • public: Este directorio almacena recursos estáticos (que no pasan por el procesamiento de compilado de Angular), como imágenes, fuentes o archivos HTML adicionales. Los archivos en esta carpeta se copian directamente al directorio de salida del compilado.
  • src: Esta es la carpeta más importante porque va a contener todos los ficheros que tenemos que modificar para darle forma a nuestro proyecto. Se componentes de una serie de carpetas y subcarpetas que a continuación comentaremos.
    • app/: Contiene los componentes, servicios, directivas y módulos que forman la aplicación.
    • assets/: Se utiliza para almacenar recursos estáticos como imágenes, íconos o archivos JSON, que pueden ser utilizados por la aplicación. Generalmente la tendremos que crear nosotros.
    • environments/: Incluye configuraciones específicas para diferentes entornos (desarrollo, producción, etc.). También puede que no se genere automáticamente.
    • index.html: El archivo principal HTML que sirve de contenedor principal para nuestro proyecto
    • main.ts: El punto de entrada de la aplicación donde Angular inicializa la aplicación. Todos los ficheros con extesión.ts son de typesrcipt
    • styles.css o styles.scss: hojas de estilo.

    Ficheros que actúan en el framework

    Los ficheros principales que gestionan Angular son: .ts, .html y .css:

  • TypeScript (.ts): Contiene la lógica del componente (controlador), como el manejo de eventos, propiedades y funciones.
  • HTML (.html): Define la plantilla (lo que se va a ver) del componente.
  • CSS o SCSS: Define los estilos específicos para ese componente.

    • Estos archivos se editan utilizando cualquier editor de texto con soporte para Angular y TypeScript. El más utilizado es Visual Code de Microsoft que permite la instalación de muchos plugins que ayudan a mejorar la programación y es un software gratuito.

    Para la generación de componentes, servicios y otros recursos Angular dispone de comandos "como ng generate component" que hemos comentado anteriormenete.

    Otros ficheros importantes en Angular son angular.json, tsconfig.json (ficheros de configuración globales) que controlan aspectos como la compilación, la estructura del proyecto y los ajustes del compilador TypeScript.

    ¿Actúa sobre páginas, ficheros, secciones…?

    Angular es un framework basado en componentes que organiza las aplicaciones en módulos y componentes reutilizables. Cada componente gestiona una sección específica de la interfaz de usuario (UI), lo que significa que no actúa sobre páginas completas, sino sobre secciones dinámicas que se combinan para construir la aplicación.

    El framework opera sobre un único archivo HTML principal (index.html) que sirve como punto de entrada. Desde allí, Angular inyecta y gestiona dinámicamente los componentes en función de las rutas, eventos o estados definidos. Esto permite crear aplicaciones de una sola página (SPA) donde la navegación entre vistas no implica recargar toda la página, sino cambiar secciones del DOM en tiempo real.

    Referencias a otras aplicaciones

    Angular permite crear bibliotecas o módulos independientes que pueden ser empaquetados y publicados en un gestor de paquetes como npm. Otras aplicaciones pueden instalarlos como dependencias y utilizarlos importándolos en sus propios módulos.

    Si se trata de APIs o servicios creados en Angular, estas aplicaciones pueden ser consumidas desde otras mediante solicitudes HTTP (HttpClient). Los servicios se definen y exponen dentro del proyecto Angular para ser accedidos desde cualquier cliente que haga las solicitudes.

    Preguntas frecuentes sobre Angular

  • ¿Qué es Angular? Angular es un framework de desarrollo web basado en TypeScript que permite crear aplicaciones de una sola página (SPA).
  • ¿Para qué sirve Angular? Angular se utiliza para construir aplicaciones web modernas, escalables y dinámicas con una arquitectura modular.
  • ¿Es Angular solo para frontend? Sí, Angular se enfoca exclusivamente en el desarrollo de interfaces de usuario (UI).
  • ¿Qué lenguaje utiliza Angular?Angular utiliza TypeScript, un superconjunto de JavaScript, para escribir la lógica de sus aplicaciones.
  • ¿Cuál es la diferencia entre AngularJS y Angular? AngularJS es la primera versión basada en JavaScript, mientras que Angular (2+) utiliza TypeScript y una arquitectura completamente renovada.
  • ¿Cómo organiza Angular una aplicación? Angular organiza las aplicaciones en módulos, componentes, servicios y rutas.
  • ¿Qué es un componente en Angular? Es una unidad reutilizable que combina lógica (TypeScript), vista (HTML) y estilos (CSS/SCSS).
  • ¿Qué es un módulo en Angular? Un módulo agrupa componentes, directivas y servicios relacionados para estructurar la aplicación.
  • ¿Qué es el Data Binding en Angular? Es la sincronización automática de datos entre la vista y el modelo de datos.
  • ¿Cómo gestiona Angular los datos compartidos? Los datos se comparten usando servicios, propiedades @Input/@Output o un estado global.
  • ¿Cómo se crea un nuevo proyecto Angular? Usando el comando ng new nombre-proyecto.
  • ¿Cómo se genera un nuevo componente? Ejecutando ng generate component nombre-componente o ng g c nombre-componente.
  • ¿Cómo se inicia el servidor de desarrollo? Con el comando ng serve.
  • ¿Cómo se compila una aplicación para producción? Usando ng build --prod.
  • ¿Cómo se genera un servicio en Angular? Mediante ng generate service nombre-servicio o ng g s nombre-servicio.
  • ¿Qué es el Angular Router? Es una herramienta que permite navegar entre vistas o componentes mediante URLs.
  • ¿Cómo se definen rutas en Angular? Se configuran en el archivo app-routing.module.ts.
  • ¿Qué es el Lazy Loading? Es una técnica para cargar módulos solo cuando son necesarios, mejorando el rendimiento.
  • ¿Cómo se depura una aplicación Angular? Usando herramientas como Chrome DevTools y extensiones como Augury para inspeccionar el estado de la aplicación.
  • ¿Qué es Ahead-of-Time (AOT)? Es una compilación previa que optimiza la aplicación antes de ejecutarse en el navegador.
  • ¿Cómo se minimiza el tamaño del paquete? Configurando el modo producción con ng build --prod y utilizando Lazy Loading.
  • ¿Qué es Angular Universal? Es una herramienta para renderizar aplicaciones Angular en el servidor, mejorando SEO y tiempo de carga.
